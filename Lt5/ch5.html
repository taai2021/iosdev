1. 使用List製作簡易清單

簡單清單

struct ContentView: View { var body: some View { List { Text("Hello, world!") .padding() Text("Hello, world!") .padding() Text("Hello, world!") .padding() Text("Hello, world!") .padding() } } }

ForEach功能的簡單清單

struct ContentView: View { var body: some View { List(0 ... 30, id:\.self){index in Text("Hello, world!") .padding() } } }

加上索引號的ForEach簡單清單

struct ContentView: View { var body: some View { List(0...30, id:\.self) { index in Text("第\(index)次 Hello, world! ") .padding() } } }

使用STRUCT+identifiable-更簡潔的方式

使用restaurants[index]畢竟還是不友善，如果能類似 傳統for loop → for each 那就更方便了 for (int index=0; index < restaurants; index++){ 使用 restaurants[index].image } 改成 for restaurant in restaurants 使用 restaurant.image 是不是親切多了 做法就是 要讓struct 遵循identifiable，並且新增一個ID當作唯一的編號 struct Restaurant:Identifiable{ var id = UUID() var name: String var image: String var location: String var type: String }

UUID()可以產生一個獨一無二的數當作ID，避免重複 ，程式碼更簡潔

// 比較一下index方式 // List(restaurants.indices, id:\.self) { index in List(restaurants) { restaurant in HStack { Image(restaurant.image) .resizable() .frame(width: 80.0, height: 80.0) .clipShape(Circle()) VStack(alignment: .leading) { Text(restaurant.name) Text(restaurant.type) Text(restaurant.location) } } }

整理一下

把要顯示的內容做成另一個view，維護比較容易，每一個餐廳的資訊以restaurant變數傳到另一個view來顯示

var body: some View { List(restaurants) { restaurant in RestView(restaurant: restaurant) } } } struct RestView: View { var restaurant:Restaurant var body: some View { HStack { Image(restaurant.image) .resizable() .frame(width: 80.0, height: 80.0) .clipShape(Circle()) VStack(alignment: .leading) { Text(restaurant.name) Text(restaurant.type) Text(restaurant.location) } } } }
