Button、Textfield以及變數綁定

struct是不可變的 不能修改 包括變數

import SwiftUI struct ContentView: View { var x="Helo, world!" var body: some View { Button(/*@START_MENU_TOKEN@*/"Button"/*@END_MENU_TOKEN@*/) { x="Hello, world!" } VStack { Text(x) .padding() } } }

binding : 用 @State 綁變數 就可以修改了

import SwiftUI struct ContentView: View { @State var x="Helo, world!" var body: some View { Button(/*@START_MENU_TOKEN@*/"Button"/*@END_MENU_TOKEN@*/) { x="Hello, world!" } VStack { Text(x) .padding() } } }

Two ways binding : 用 @State 綁變數 就可以修改了

前面的binding是單向，修改變數，然後元件上的也被改變，
但是有些應用需要能在元件上數值改變之後，binding的變數也跟著改，例如Textfield，
當使用者在TextField上輸入文字之後，變數也會跟著改變，也就是雙向改變
做法是在需要雙向的地方加上$，例如下方程式y綁到textfield，需要雙向所以加上$
而Button裡面的 x=y 並不需要雙向改變所以使用y，而不是$y

import SwiftUI struct ContentView: View { @State var y="" @State var x="Helo, world!" var body: some View { TextField(/*@START_MENU_TOKEN@*/"Placeholder"/*@END_MENU_TOKEN@*/, text: $y) Button(/*@START_MENU_TOKEN@*/"Button"/*@END_MENU_TOKEN@*/) { x=y } VStack { Text(x) .padding() } } }

binding struct: 一次把多個變數綁起來，用 @State 綁進來使用

應該可以發現下面程式，按鈕用不著了，因為會自動更新

import SwiftUI struct rest{ var name="饗食天堂" var location="台中市大遠百" var type="歐式自助" } struct ContentView: View { @State var y=rest() var body: some View { VStack { TextField("輸入提示", text: $y.name) TextField("輸入提示", text: $y.location) TextField("輸入提示", text: $y.type) Button("Button") { //不用做什麼 } Text(y.name) Text(y.location) Text(y.type) } } }

因為共用y 所以會一起變化，假設你需要分開變化呢?等到你的指令再更新TEXT的資料?
那麼多宣告一個x，y給 textfield用，x給TEXT用，如下

import SwiftUI struct rest{ var name="饗食天堂" var location="台中市大遠百" var type="歐式自助" } struct ContentView: View { @State var y=rest() @State var x=rest() //加一個x var body: some View { VStack { TextField(/*@START_MENU_TOKEN@*/"Placeholder"/*@END_MENU_TOKEN@*/, text: $y.name) TextField(/*@START_MENU_TOKEN@*/"Placeholder"/*@END_MENU_TOKEN@*/, text: $y.location) TextField(/*@START_MENU_TOKEN@*/"Placeholder"/*@END_MENU_TOKEN@*/, text: $y.type) Button(/*@START_MENU_TOKEN@*/"Button"/*@END_MENU_TOKEN@*/) { } Text(y.name) Text(y.location) Text(y.type) } } }
