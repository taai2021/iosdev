2. tuples 元組

元組（tuples）把多個值組合成一個複合值。元組內的值可以是任意類型，並不要求是相同類型。 下面這個例子中，(404, "Not Found")是一個描述HTTP狀態碼（HTTP status code）的元組。 HTTP狀態碼是當你請求網頁的時候web服務器返回的一個特殊值。如果你請求的網頁不存在就會返回一個404 Not Found狀態碼。 let http404Error = (404, "Not Found") // http404Error 的类型是 (Int, String)，值是 (404, "Not Found") (404, "Not Found")元組把一個Int值和一個String值組合起來表示HTTP狀態碼的兩個部分： 一個數字和一個人類可讀的描述。這個元組可以被描述為“一個類型為(Int, String)的元組”。 let complex = (1.0, -2.0) // 一個 (Double, Double) 的tuple let (real, imag) = complex //可以這樣拆解 let (real, _) = complex // 底線代表忽略那個值  使用索引值存取  let real = complex.0 let imag = complex.1   也可以幫元素命名 let complex = (real: 1.0, imag: -2.0) let real = complex.real

練習

3. optionals

nil是指空值，對應到java是null， 在swift， var a:int=1 如果設定a=nil 就會出現錯誤 請大家試試看 var a:Int = 1 a=100 //沒問題 a = nil // 失敗 nil是指空值，什麼都沒有，連0都沒有 你怎麼表示a裡面什麼都沒有？ a是一個放數字的記憶體空間，不管怎樣裡面就是有010101的資料， 怎樣才能表示a變數是什麼都沒有的空值nil? swift使用可選類型（optionals）來處理值可能為nil的情況。 optional就是在原來的資料形態後面加一個？ 例如 var b:Int? = 1
b = nil // 成功 Int？ 就是Int的optional變數 所以 b = nil // 成功 a和b有什麼不同？也就是Int和Int?有什麼不同？ 可以這樣想像 Int? 的變數加了一個框，把裡面東西拿走，就變成空的狀態 所以可以表示nil var a:Int = 1
var b:Int? =1
當b=nil，b就變成一個空的框 b=
a就無法表示nil （Int 無法表示 空）
