使用CocoaPods管理Swift的類別程式庫

在XCODE中建置app，你常常會需要安裝套件
例如想要寫一個app和facebook互動，那你就必須安裝其SDK。
你當然可以直接下載，然後在其XCODE專案中直接一一設定。
但是如果你要加入一個套件，這個套件又會用到其他你未安裝的程式庫，
那你安裝一個，發現又相依到另一個，就必須再去抓那一個，
萬一這一個又需要其他10個，那就會有點崩潰了。套件常常具有相依性。
抓起來有時很燒心。cocoaPod就是一個可以讓你比較方便的抓抓幫手。
你只要指定你要抓那一個，其餘和這個相依的會自動全部抓回來。

在這邊將展現一個範例，展示一個專案如何加入firebase類別庫，下面是步驟

  1. 建立專案

    2. 建立一個viewControlller專案。命名為uppic

  2. 安裝cocoapods

    3. 開終端機：啟動台 → 其他 → 終端機 輸入 sudo gem install cocoapods

  3. 在終端機中，跳到uppic專案資料夾
    4. cd ~/Desktop/uppic 跳到xcode專案，我的xcode專案放在桌面，專案名稱是uppic
  4. 在uppic資料夾產生Podfile
    5. pod init 這樣會產生一個Podfile，內容如下 # Uncomment this line to define a global platform for your project # platform :ios, '9.0' target 'CocoapodsTest' do # Comment this line if you're not using Swift and don't want to use dynamic frameworks use_frameworks! # Pods for CocoapodsTest end
  5. 使用文字編輯器把內容修改如下
    6. # Uncomment this line to define a global platform for your project # platform :ios, '9.0' target 'CocoapodsTest' do # Comment this line if you're not using Swift and don't want to use dynamic frameworks use_frameworks! # Pods for CocoapodsTest pod 'Firebase' end

    pod 'Firebase使 CocoaPods 知道我們需要用上 ‘Firebase’

  6. pod install

    7. 存檔離開 終端機執行 pod install 這樣就會把Firebase套件及其相依的套件裝到xcode專案上去。

Firebase是甚麼? Firebase首頁連結

Firebase 是一個同時支援 Android、iOS 及網頁的 app 雲端開發平台， 協助 app 開發者在雲端快速建置後端服務，提供即時資料庫，有效縮短 app開發時間， 並幫助開發者更專注在前端的優化

強大的無限量數據分析功能正是 Firebase 的產品核心， 目前是全免費使用，有預設的 500 種事件類型，提供無上限的事件記錄， 還可以自定義關鍵事件以及使用者屬性， 視覺化的的儀表板（Dashboard）讓使用者一眼看到所有最重要的使用者行為和行銷活動的成效分析。

三大功能

  1. 開發 (Develop)

    2. 使用 Firebase 打造高品質的 app，安全且高效能，提供的後端服務包含即時資料庫（Realtime database）、 簡單的身分驗證模組（Authentication）、雲端訊息（Cloud Messaging）、主機（Hosting）、 檔案儲存（Storage）、及遠端配置（Remote Config）， 並提供 Android 測試實驗室（Test Lab for Android）以及當機報告（Crash Reporting） 等讓開發者可以輕鬆測試 app 在不同的 OS 與硬體裝置下的表現，快速發現問題並確保 app 運行品質。

  2. 成長（Grow）

    3. Firebase 讓開發者可以輕鬆設定訊息推播系統 （notifications）、 app 索引功能（App Indexing）、動態連結（Dynamic Links）、 email 及 sms 邀請功能（Invites）、以及與 Google AdWords 的深度整合串接 app 行為資料， 讓 app 開發者能提供更精準的廣告或溝通訊息給對的目標族群。

  3. 獲利（Earn)

    4. 與自家產品 AdMob 整合，加入廣告聯播網，輕鬆放入廣告版位， 以及多元化的廣告格式，以及更清楚的追蹤機制，幫助 app開發者輕鬆獲利。
